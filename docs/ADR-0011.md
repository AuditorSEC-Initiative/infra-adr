## Context

AuditorSEC Initiative operates under the **NIS2 Directive (EU) 2022/2555** and Ukrainian cybersecurity legislation. As part of UHIP-2A (Unified Hybrid Infrastructure Programme Phase 2A), all AI/LLM tools used in infrastructure, legal, and security-sensitive workflows must be assessed against NIS2 Art.21 risk management obligations.

Grok (by xAI / X Corp) was identified as a high-risk external AI chatbot due to:

1. **Data residency**: xAI operates outside EU/UA jurisdiction with no EU Standard Contractual Clauses (SCC) or GDPR Art.46 transfer mechanisms confirmed.
2. **Supply chain risk**: xAI is a US-based vendor without a published Data Processing Agreement (DPA) compliant with GDPR/NIS2 requirements.
3. **Post-quantum cryptography**: No evidence of PQ-safe encryption for data in transit/at rest.
4. **Human resources security**: Social engineering risk via Grok's unrestricted conversation history access.
5. **BRAVE1 compatibility**: Ukrainian BRAVE1 security clearance requirements are incompatible with xAI's current data governance posture.

---

## Decision

**PROHIBIT** the use of Grok/xAI in all AuditorSEC NIS2-scoped, BRAVE1-classified, and UHIP-2A workloads.

This prohibition applies to:
- All team members under AuditorSEC-Initiative org
- All automated pipelines (n8n, Zapier, CI/CD)
- All infrastructure management tasks (Vault, ArgoCD, Kubernetes)
- All legal/compliance document processing

### Enforcement Mechanism

```yaml
# governance.yaml (this repo)
aiChatbots:
  grok:
    status: PROHIBITED
    enforcement: BLOCK at network perimeter (Woodpecker + ArgoCD)
```

Woodpecker CI will lint `governance.yaml` on every PR. ArgoCD will enforce at runtime via network policy.

## Consequences

### Positive
- Full NIS2 Art.21 compliance for AI tooling
- Reduced supply chain risk (Art.21.2d)
- BRAVE1 clearance maintained
- Audit trail via n8n + S3 immutable logs

### Negative
- Team members must migrate Grok-based workflows to Ollama/Perplexity
- Initial overhead for Ollama K8s deployment (see deployment plan below)

## Countermeasures (Kontrzakhody)

### 1. Ollama Deployment (Mistral Nemo PQ)

```bash
# Namespace
kubectl create ns ollama-system

# Deploy via Helm + ArgoCD
helm install ollama otwld/ollama \
  -f values-pq.yaml \
  --set models=mistral-nemo-pq \
  --set image=ollama/ollama:latest-pq \
  -n ollama-system
```

Monitor: GPU util >90% alert, target 120 t/s on RTX4090 equiv.
Stack: Grafana + Prometheus

### 2. n8n Monitoring Workflow (Google Dorks)

Cron trigger → Google dorks:
- `"auditorsec" site:grok.com/share`
- `"audityzer" site:grok.com/share`

Flow: Parsera scrape → Grafana alert + S3 log (immutable via `make ai-audit-logs`)

### 3. Vault Encryption

All chat logs encrypted locally via Vault. Key management: HashiCorp Vault + ArgoCD sync.
